# 便签应用 · AI 轻量增强功能需求文档

> 文档版本：V1.1  
> 更新日期：2026-07-16  
> 核心理念：**主动触发 · 单条上下文 · 时间硬边界 · 零打扰**


## 一、设计原则（必须遵守）

| 原则 | 说明 |
| :--- | :--- |
| **主动触发** | 所有AI功能必须通过「右键菜单」「工具栏按钮」「输入框旁图标」显式唤起，**禁止自动弹窗、自动改写、定时推送** |
| **单条隔离** | 除「周报/月报」外，AI仅读取「当前选中的文本」或「当前编辑的单条便签」，**不扫描历史笔记、不做全局语义检索** |
| **时间硬边界** | 周报/月报仅汇总「最近7天/30天」的便签，上限20条，**绝不全量遍历** |
| **即改即显** | AI返回结果后，直接替换选中文本或插入光标处/新建便签，不另开对话窗口 |
| **离线降级** | 未配置API Key时，所有AI入口隐藏或置灰，应用原有纯本地功能完全不受影响 |
| **轻量Token** | 单次请求上下文 ≤ 2000字符，输出 ≤ 500字符，保证秒级响应 |
| **可撤销** | 所有AI改写/插入操作均进入撤销栈，支持 `Ctrl+Z` 回退 |


## 二、功能模块详情

### 模块1：智能文本规整（内容清洗）

**位置**：编辑区右键菜单 / 选中文字后的悬浮工具栏  
**适用场景**：随手记的草稿太乱，一键变工整

| 子功能 | 输入示例 | 输出示例 | 交互路径 |
| :--- | :--- | :--- | :--- |
| **口语转书面** | "周四下午找老王对一下那个项目的预算和排期" | "周四下午与老王对齐项目预算及排期。" | 选中 → 右键 → "✨ 规整文本" |
| **一键转待办** | "买牛奶、交水电费、回邮件" | `- [ ] 买牛奶`<br>`- [ ] 交水电费`<br>`- [ ] 回邮件` | 选中 → 右键 → "📋 转为清单" |
| **去冗余精简** | "我觉得这个功能其实可能大概也许可以加上" | "建议加上此功能。" | 选中 → 右键 → "✂️ 精简表达" |

**技术约束**：
- 仅当选中文本长度 **5~500 字符** 时启用，超出提示"请分段处理"
- Prompt 固定模板：`将以下文本执行[规整/转清单/精简]操作，仅返回结果，不附加任何解释：{选中文本}`


### 模块2：智能表单填充（嗅探辅助）

**位置**：便签底部「添加提醒」按钮旁 / 保存便签时的标签输入框旁  
**适用场景**：减少手动翻日历、敲标签的步骤

| 子功能 | 触发时机 | 行为 |
| :--- | :--- | :--- |
| **⚡ 提醒嗅探气泡** | 便签正文含时间关键词（如"下周三""明天上午""腊月廿三"） | 在提醒设置区自动显示气泡：`检测到"下周三"，一键添加提醒？`，点击后自动填充日期/时间/标题。**需用户确认后才保存** |
| **🏷️ 标签速配** | 点击保存便签时（或标签框旁的AI图标） | 基于**当前便签全文**推荐最多3个标签（如 `#工作` `#bug` `#会议`），以彩色标签形式展示，点击即添加。**不自动写入** |

**约束**：
- 提醒嗅探仅识别自然语言日期，结合 `tyme4rs` 支持农历/节气（如"腊月廿三""立春"）
- 标签推荐基于当前便签词频，不做跨便签语义分析


### 模块3：待办清单智能排序

**位置**：当便签内包含 `- [ ]` 清单时，清单区域顶部出现排序按钮  
**适用场景**：随手列了5件事，按紧急程度排个序

- **操作**：点击清单上方的 `🔽 AI 排序（按紧急度）`
- **逻辑**：AI读取当前便签内所有 `- [ ]` 条目，根据关键词（"紧急""deadline""明天""必须"）重新排列
- **降级**：如果条目 ≤ 3项，按钮置灰（没必要AI介入）
- **反馈**：排序过程显示轻量加载动画（≤1秒），排序后自动保存便签


### 模块4：文风切换（单条重写）

**位置**：选中文本后的右键菜单二级菜单  
**适用场景**：把笔记内容复制发给老板/同事/客户时快速改变语气

| 选项 | 效果 | 压缩比 |
| :--- | :--- | :--- |
| `👔 更正式` | 口语→书面，添加敬语，去除个人情绪词 | — |
| `📢 更精简` | 压缩字数，只留核心信息 | 原字数的 60% |
| `🤝 更温和` | 强硬表述（"必须""错了"）→建议式（"建议""可优化"） | — |

- **交互**：选中 → 右键 → 选择风格 → 原文被替换（支持 `Ctrl+Z` 撤销）
- **限制**：仅当选中文本长度 **10~500 字符** 时生效


### 模块5：自然语言创建提醒（专属增强）

**位置**：点击「新建提醒」弹出的对话框内（独立于便签正文）  
**适用场景**：用户不想在日期控件里点来点去

- **输入框**：在提醒弹窗中增加文本输入框，占位符：`试试说"从明天起每天早8点提醒吃药"`
- **解析字段**（必须全部提取）：
  - `title`（提醒标题）
  - `start_time`（首次触发时间，精确到分钟）
  - `repeat_type`：`once / daily / weekly / monthly / lunar_monthly`
  - （若为 `monthly` 或 `lunar_monthly`，需提取日期数，如"每月15号"）
- **结合 tyme4rs**：支持"腊月廿三""明年立春"等农历/节气表达，映射到公历日期
- **兜底**：若AI解析失败或置信度低，自动回退到传统表单模式，并提示"未能识别，请手动填写"


### 模块6：周报/月报自动生成（轻量回顾）

**定位**：不是"AI替你写作文"，而是"AI帮你把本周零散记录整理成清单，稍作修改就能发出去"

#### 触发入口（3个位置）

| 位置 | 交互 |
| :--- | :--- |
| **日历视图 - 月视图顶部** | 按钮 `📊 生成月报`（仅当本月有 ≥3 条便签时亮起） |
| **侧边栏底部** | 快捷入口 `📋 生成周报`（默认汇总上周一~周日） |
| **右键菜单（日期格子）** | 右键点击日历某天 → `生成截至当天的周报` |

#### 数据拾取范围（硬边界）

| 数据类型 | 说明 | 上限 |
| :--- | :--- | :--- |
| 本周/本月新建/编辑过的便签 | 取标题 + 正文（截取前200字/条） | **20条** |
| 待办完成情况 | 统计：已完成数 / 总数，列出已完成待办标题 | — |
| 提醒触发记录 | 时间段内触发过的提醒标题（默认不包含，可选） | — |

> ⚠️ 拾取总数上限 **20条便签**，超出按更新时间倒序取最近20条，保证请求体 ≤ 2000 字符。

#### AI 输出格式（结构化 Markdown）

返回内容直接**插入为一个新便签**（标题自动带日期，如 `2026-07-13 ~ 07-19 周报`），结构固定为：

```markdown
## 📌 本周/本月重点（3-5 条摘要）
- [AI提炼的核心点]

## ✅ 已完成事项
- [从待办中提取的已完成项]

## ⏳ 进行中 / 待推进
- [未完成待办 + 便签中识别出的未完结事项]

## 💡 零散记录
- [其他便签的简要罗列]
```

#### 交互流程

1. 用户点击「生成周报」/「生成月报」
2. 弹出轻量进度提示 ⏳ 正在汇总...
3. 拾取数据 → 调用 AI
4. 返回结果 → 自动新建便签，标题预填，内容为AI生成的草稿
5. 自动打开编辑模式，用户可自由修改/删减后 Ctrl+S 保存

#### 与"知识库RAG"的本质区别

| 对比项 | 知识库RAG（不做） | 本模块（做） |
| :--- | :--- | :--- |
| 数据范围 | 全部历史，无时间限制 | 仅最近 7/30 天，硬边界 |
| 检索方式 | 语义相似度匹配 | 时间倒序枚举，无检索 |
| 输出目的 | 回答问题 / 深度分析 | 整理罗列，不添加主观推断 |
| 数据量 | 成千上万条 | 上限 20 条 |


## 三、交互与视觉规范

| 元素 | 规范 |
| :--- | :--- |
| AI功能入口图标 | 统一使用 ✨ 或 ⚡ 前缀，颜色为琥珀色（#F59E0B），区别于普通功能 |
| 加载状态 | AI处理时，按钮变为"⏳ 处理中..."，禁止重复点击 |
| 错误提示 | API超时/报错时，toast提示"AI服务暂时不可用，请检查网络或Key配置"，不影响原有编辑内容 |
| 操作记录 | 所有AI改写/插入操作均进入撤销栈（Ctrl+Z 可回退） |
| 周报新建便签 | 自动使用默认配色（或用户设定的"周报专用色"），标题前缀 [周报] / [月报] 便于识别 |


## 四、技术约束与配置

| 项目 | 要求 |
| :--- | :--- |
| API支持 | 默认支持 OpenAI / Claude / DeepSeek（用户自填 Base URL 和 Key），可扩展 |
| 本地存储 | API Key 仅保存在本地配置文件（不随 Git 同步） |
| Token限制 | 单次请求 max_tokens=500，temperature=0.3（保证输出稳定） |
| 超时时间 | 5秒，超时则报错降级 |
| 请求体上限 | 单次请求上下文 ≤ 2000 字符 |
| Prompt管理 | 所有 Prompt 采用"系统指令 + 用户输入"固定格式，统一存放在 `src-tauri/src/application/prompts/` 目录 |
| 周报数据拾取 | 仅读取本地数据库（SQLite），按 updated_at 倒序过滤时间窗口 |
| API Key 配置入口 | Hub 设置中心 → 「AI 配置」页面：Base URL、API Key、模型名 |

### AI 配置页面（Hub 新增）

| 字段 | 说明 | 默认值 |
| :--- | :--- | :--- |
| Base URL | API 端点地址 | `https://api.openai.com/v1` |
| API Key | 用户密钥，本地加密存储 | 空（未配置时所有AI入口置灰） |
| 模型名 | 使用的模型 | `gpt-4o-mini` |
| 测试连接 | 点击按钮验证配置是否正确 | — |

### 架构设计

```
application/
├── ai_service.rs       # AI 调用封装（HTTP client、重试、超时、错误处理）
├── ai_config.rs        # AI 配置读写（Base URL / Key / Model）
└── prompts/
    ├── mod.rs          # Prompt 模板注册
    ├── tidy.rs         # 模块1：规整/转清单/精简
    ├── style.rs        # 模块4：文风切换
    ├── reminder.rs     # 模块5：自然语言提醒解析
    ├── sort.rs         # 模块3：待办排序
    └── report.rs       # 模块6：周报/月报生成
```

**依赖方向**：前端 → `commands.rs` → `ai_service.rs` → 远程 API  
**domain 层不引入 AI 依赖**，AI 能力完全在 application 层实现。


## 五、明确不做（边界声明）

以下需求本次明确排除，防止范围蔓延：

- ❌ 基于历史笔记的全局语义搜索 / 向量检索（RAG）
- ❌ 跨便签的自动分类、归档建议
- ❌ 聊天式问答窗口 / 对话历史
- ❌ 云端模型训练或微调
- ❌ 定时自动生成周报/月报（每月1号/每周一自动弹窗）
- ❌ 情感分析、长期趋势统计、知识图谱构建
- ❌ 跨用户的数据聚合或协作
- ❌ 流式输出（SSE）——首版用同步请求，简化实现


## 六、排期建议（按MVP优先级）

| 阶段 | 模块 | 依赖 | 理由 |
| :--- | :--- | :--- | :--- |
| **P0（首版必做）** | AI 基础设施：配置页面 + ai_service + prompts 框架 | 无 | 所有模块的前置 |
| **P0** | 模块5：自然语言创建提醒 | AI基础设施 + 现有提醒表单 | 体验提升最直观，用户感知最强 |
| **P0** | 模块2-嗅探：提醒嗅探气泡 | 模块5的日期解析能力 | 复用解析能力，锦上添花 |
| **P1（第二版）** | 模块1：智能规整（口语→书面 + 转清单 + 精简） | AI基础设施 | 右键菜单扩展，高频场景 |
| **P1** | 模块4：文风切换（正式/精简/温和） | 复用模块1 Prompt 工程 | 与模块1共享改写逻辑 |
| **P1** | 模块6：周报/月报自动生成 | 日历视图 + 便签列表查询接口 | 数据拾取逻辑稍复杂 |
| **P2（后续迭代）** | 模块3：待办清单智能排序 | 无 | 条件触发，低频 |
| **P2** | 模块2-标签：标签速配 | 现有标签体系 | 简单但价值有限 |


## 七、附录：API 请求/响应示例

### 自然语言创建提醒（模块5）

请求：

```json
{
  "model": "gpt-4o-mini",
  "messages": [
    {
      "role": "system",
      "content": "解析用户输入的提醒语句，返回JSON格式。字段：title(字符串), start_time(YYYY-MM-DD HH:mm), repeat_type(once/daily/weekly/monthly/lunar_monthly), repeat_day(数字，仅monthly/lunar_monthly需要)。仅返回JSON。"
    },
    {
      "role": "user",
      "content": "从后天开始，每周五下午5点提醒我发周报"
    }
  ],
  "temperature": 0.1,
  "max_tokens": 150
}
```

响应：

```json
{
  "title": "发周报",
  "start_time": "2026-07-18 17:00",
  "repeat_type": "weekly",
  "repeat_day": 5
}
```

### 周报生成（模块6）

请求（简化示例）：

```json
{
  "model": "gpt-4o-mini",
  "messages": [
    {
      "role": "system",
      "content": "根据以下便签列表生成周报草稿，按四个板块输出Markdown：## 📌 重点、## ✅ 已完成、## ⏳ 进行中、## 💡 零散记录。不添加额外解释。"
    },
    {
      "role": "user",
      "content": "便签列表：\n1. [2026-07-14] 修复了登录页面的bug\n2. [2026-07-15] - [ ] 完成项目排期表\n3. [2026-07-16] 与老王讨论了新需求\n4. [2026-07-17] - [x] 提交季度报告"
    }
  ],
  "temperature": 0.3,
  "max_tokens": 500
}
```

响应（直接插入为便签内容）：

```markdown
## 📌 本周重点
- 修复登录页bug，已完成
- 与老王对齐新需求，待排期

## ✅ 已完成事项
- 提交季度报告

## ⏳ 进行中 / 待推进
- 完成项目排期表

## 💡 零散记录
- 7月14日修复登录页bug
- 7月16日与老王讨论新需求
```
