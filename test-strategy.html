一、测试分层架构

项目遵循 DDD 四层架构，测试按层级分为四类，从底到顶逐级递进：

第一层
单元测试
domain 层纯逻辑
→
第二层
集成测试
infrastructure 层 SQLite
→
第三层
应用层测试
application 层 IPC 命令
→
第四层
端到端测试
Tauri 窗口 + IPC + UI

1. 单元测试（Unit Test）

目标：验证 domain 层的纯业务逻辑，不依赖任何外部资源。

范围：

文件测试内容数量状态
domain/note.rs创建、更新内容、透明度边界、置顶、颜色、窗口状态、标题更新8已完成
domain/reminder.rs创建、到期判断、贪睡、每日/每周/每月重复、标记完成、取消8已完成
domain/value_objects.rs颜色解析、CSS 值、窗口默认值3已完成

工具：Rust 内置 #[test] + cargo test

特点：无外部依赖，执行速度 <1ms/测试，可并行运行

2. 集成测试（Integration Test）

目标：验证 infrastructure 层 SQLite 仓储的 CRUD 操作和数据库迁移。

范围：

文件测试内容数量状态
sqlite_note_repo.rs保存查询、批量查询、排序、更新内容/样式/窗口状态、删除、不存在查询8已完成
sqlite_reminder_repo.rs保存查询、到期查询、未到期过滤、状态更新、贪睡、删除6已完成
database.rs内存数据库创建、WAL 模式验证2已完成

工具：Rust #[cfg(test)] + SQLite :memory: 内存数据库

特点：每个测试独立创建内存数据库，互不干扰，执行速度 <10ms/测试

3. 应用层测试（Application Test）

目标：验证 application 层的 IPC 命令和调度器逻辑。

范围：

模块测试内容状态
commands.rscreate_note、get_note、update_note_content 等 12 个 IPC 命令待开发
reminder_scheduler.rs到期提醒轮询、通知发送、周期提醒重置待开发

工具：tauri::test + Mock State

难点：Tauri 命令需要 AppHandle 和 State，需用 tauri::test::mock_app 构建 Mock 环境

4. 端到端测试（E2E Test）

目标：验证完整用户流程——从托盘点击到窗口创建、编辑、保存、提醒触发。

范围：

场景验证点状态
新建便签流程托盘双击 → 窗口出现 → 编辑内容 → 关闭 → 重启恢复待开发
颜色/透明度切换点击颜色 → 背景变化 → 拖滑块 → 透明度变化 → 重启保留待开发
提醒触发设置提醒 → 等待到期 → 系统通知弹出 → 贪睡/关闭待开发
多窗口管理创建多个便签 → 独立移动/关闭 → 互不干扰待开发

工具：tauri-driver + WebDriverIO 或 Playwright

特点：需要完整的 WebView2 环境，执行慢（秒级），适合发版前运行

二、执行时机策略

什么时候做什么测试？

触发场景执行的测试耗时阻塞/非阻塞
每次保存文件
IDE 自动触发		 cargo check（编译检查） ~5s 非阻塞
提交代码前
git pre-commit hook		 单元测试 + 集成测试
cargo test --lib		 ~30s 阻塞
推送到远程
git pre-push hook		 单元 + 集成 + 应用层测试 ~1min 阻塞
合并到主分支
CI/CD Pipeline		 全量回归测试（单元+集成+应用层）+ 构建 ~3min 阻塞
发版前
手动触发		 全回归 + 端到端测试 + 手动验收 ~15min 阻塞
关键功能变更后
如窗口管理、提醒系统		 对应模块的 E2E 测试 ~5min 建议

三、具体命令

日常开发

# 编译检查（最快，不运行测试）
cargo check

# 运行所有单元测试 + 集成测试
cargo test

# 只运行 domain 层测试
cargo test domain

# 只运行 infrastructure 层测试
cargo test infrastructure

# 运行单个测试
cargo test test_save_and_find_by_id

# 显示 println 输出
cargo test -- --nocapture

# 生成测试覆盖率报告（需安装 tarpaulin）
cargo tarpaulin --out Html

构建前验证

# 完整验证：测试 + 构建
cargo test && npx tauri build --no-bundle

端到端测试（待实现）

# 安装 tauri-driver
cargo install tauri-driver

# 启动 WebDriver（需 ChromeDriver）
chromedriver --port=4444

# 运行 E2E 测试
npx playwright test

五、Git Hook 配置

pre-commit（提交前运行单元+集成测试）

# .git/hooks/pre-commit
#!/bin/sh
cd src-tauri && cargo test --lib 2>&1
if [ $? -ne 0 ]; then
  echo "❌ 测试未通过，提交被阻止"
  exit 1
fi
echo "✅ 全部测试通过"
exit 0

pre-push（推送前增加应用层测试）

# .git/hooks/pre-push
#!/bin/sh
cd src-tauri && cargo test 2>&1
if [ $? -ne 0 ]; then
  echo "❌ 测试未通过，推送被阻止"
  exit 1
fi
echo "✅ 全部测试通过，允许推送"
exit 0
